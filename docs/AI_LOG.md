# AI 辅助开发日志

## 1. 本次使用 AI 辅助编程的目标

基于当前目录已有的 Win32/GDI 坦克大战项目继续开发，在不推倒重写的前提下，完善地形、道具、敌人、Boss、四关卡、UI 与文档，并保持 Visual Studio 2022 可直接编译运行。
在此基础上，本轮继续将项目从“功能完成”推进到“界面统一、图标统一、菜单完整、演示效果更好”的提交状态。

## 4. 发现的问题

- 当前项目已经具备 Win32 窗口、地图、坦克、子弹、道具与基础敌人系统。
- 现有实现已经能通过 `Debug|x64` 与 `Release|x64` 编译，但很多需求仍是简化实现。
- 现有游戏逻辑中关卡、Boss 技能、寻路、战壕状态、暂停/过关流程、统一伤害系统仍需系统化重构。
- 现有 `README.md` 已包含设置界面说明，后续更新时需要保留原有设置内容。
- 扩展后的内建自检一度出现误报：为 `LaserGuard` 和二关 Boss 搭建的测试场景不够开阔，导致技能验证结果不稳定。
- `Field Guide` 在展示项增加后出现过局部越界和重叠，需要继续压缩文案、调整卡片尺寸和底部布局。
- 旧版 HUD 虽然信息全，但观感偏向 debug overlay，不够像正式课程大作业成品。

## 5. 修正措施

- 先保留现有工程和类骨架。
- 以小步重构方式完善 `Types`、`GameMap`、`Game`、`Tank`、`Effect`、`WinApp` 等现有模块。
- 每完成一组功能后使用 MSBuild 编译 `Debug|x64` 与关键阶段构建进行验证。
- 将内建自检改为优先搜索“可视直线”和“十字开阔区”后再验证高级敌人和 Boss 技能，避免测试场景本身造成假失败。
- 把 `LaserGuard` 调整为每次技能触发都释放激光，看见玩家时优先转向玩家，以增强行为辨识度并对齐题目要求。
- 将 `Kamikaze` 的蓄力阈值修正为更接近 2 秒的时长。
- 调整 Win32 提示文案和地雷图标绘制，使按键说明与题目要求和实际实现保持一致。
- 按最新题意把第五种道具统一为“地雷”，并继续收尾 UI 表现、激光炮口和敌人移动策略。
- 将菜单、Legend、地图绘制、HUD 中重复的像素图标代码抽离到 `IconRenderer`。
- 把主菜单改为“左主视觉 + 右 Mission Briefing + 底部操作说明条”的结构，并加强按钮选中态。
- 将 HUD 改为左右分组信息块，隐藏默认 FPS，加入道具图标库存和 Boss 信息区。
- 对 `Field Guide` 的中栏与底栏进行了多轮越界/重叠修正。
- 在菜单系统继续补了 `Credits` 页面和更像正式游戏的生命格 HUD，以增强答辩展示完整性。
- 为降低帮助页局部越界风险，继续把 `Field Guide` 的列宽和底部返回区从固定坐标改为相对布局。
- 修正了一个 UI 完整性问题：`Grid Overlay` 之前只是设置项，现在已经能真实控制地图网格叠加显示。
